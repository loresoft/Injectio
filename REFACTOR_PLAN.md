# Refactor Plan: Embedded Attributes via Generator Post-Init

Move the `Injectio.Attributes` runtime assembly into source emitted by the generator using `RegisterPostInitializationOutput` + `AddEmbeddedAttributeDefinition`. The result: a single dev-only NuGet (`Injectio`) ships an analyzer that injects attribute definitions and decoration helpers directly into the consuming compilation. No runtime DLL.

---

## 0. Prerequisites — bump Roslyn floor

This refactor relies on `IncrementalGeneratorPostInitializationContext.AddEmbeddedAttributeDefinition`. Pin the Roslyn floor to **4.14.0** (Visual Studio 17.14+, .NET SDK 9.0.300+).

- `Directory.Packages.props` — change `Microsoft.CodeAnalysis.CSharp` from `[4.8.0]` to `[4.14.0]`.
- `src/Injectio.Generators/Injectio.Generators.csproj` — keep the floating `PackageReference`; only the central pin moves.

### `src/Injectio.Shared/Attributes.cs` — done

- `[System.Diagnostics.Conditional("REGISTER_SERVICE_USAGES")]` still present — kept so the `Injectio.Shared` project (which compiles these as real types) can consume them without triggering usage-site warnings. The `[Embedded]` attribute handles hiding when emitted by the generator.

## 3. Emit the embedded sources via `RegisterPostInitializationOutput`

In `src/Injectio.Generators/ServiceRegistrationGenerator.cs`, at the top of `Initialize`:

```csharp
context.RegisterPostInitializationOutput(static ctx =>
{
    ctx.AddEmbeddedAttributeDefinition();
    ctx.AddSource(
        "Injectio.Attributes.g.cs",
        SourceText.From(LoadEmbedded("Attributes.cs"), Encoding.UTF8));
    ctx.AddSource(
        "Injectio.Extensions.g.cs",
        SourceText.From(LoadEmbedded("Extensions.cs"), Encoding.UTF8));
});
```

Add a private helper:

```csharp
private static string LoadEmbedded(string name)
{
    var assembly = typeof(ServiceRegistrationGenerator).Assembly;
    var resourceName = "Injectio.Generators.Embedded." + name;
    using var stream = assembly.GetManifestResourceStream(resourceName)
        ?? throw new InvalidOperationException($"Embedded resource not found: {resourceName}");
    using var reader = new StreamReader(stream);
    return reader.ReadToEnd();
}
```

## 5. Repackage `Injectio.Generators` as the dev-only NuGet

Replace the contents of `src/Injectio.Generators/Injectio.Generators.csproj` with:

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <TargetFramework>netstandard2.0</TargetFramework>
    <IsRoslynComponent>true</IsRoslynComponent>
    <IsPackable>true</IsPackable>
    <PackageId>Injectio</PackageId>
    <DevelopmentDependency>true</DevelopmentDependency>
    <IncludeBuildOutput>false</IncludeBuildOutput>
    <SuppressDependenciesWhenPacking>true</SuppressDependenciesWhenPacking>
    <AnalyzerLanguage>cs</AnalyzerLanguage>
    <EnforceExtendedAnalyzerRules>true</EnforceExtendedAnalyzerRules>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.CodeAnalysis.CSharp" PrivateAssets="all" />
  </ItemGroup>

  <ItemGroup>
    <EmbeddedResource Include="..\Injectio.Shared\Attributes.cs"
                      LogicalName="Injectio.Generators.Embedded.Attributes.cs" />
    <EmbeddedResource Include="..\Injectio.Shared\Extensions.cs"
                      LogicalName="Injectio.Generators.Embedded.Extensions.cs" />
  </ItemGroup>

  <ItemGroup>
    <None Include="$(OutputPath)\$(AssemblyName).dll"
          PackagePath="analyzers/dotnet/roslyn4.14/cs"
          Pack="true" Visible="false" />
    <None Include="Injectio.targets"
          PackagePath="build/$(PackageId).targets"
          Pack="true" Visible="false" />
  </ItemGroup>

</Project>
```

Notes:

- `IncludeBuildOutput=false` + `DevelopmentDependency=true` keep the package out of consumers' runtime closures.
- `SuppressDependenciesWhenPacking=true` ensures no transitive dependency on `Microsoft.CodeAnalysis.CSharp` is published.
- The `build/$(PackageId).targets` rename ensures NuGet auto-imports the targets file when the package is installed.
- `analyzers/dotnet/roslyn4.14/cs` matches the new minimum Roslyn floor (4.14.0).

## 10. Suggested commit sequencing

Land in this order so each step is independently buildable and snapshot churn is isolated to one commit:

1. **Wire shared sources as embedded resources, no behaviour change yet.** The `src/Injectio.Shared/Attributes.cs` and `src/Injectio.Shared/Extensions.cs` files are already complete. Add the `EmbeddedResource` items to the generator project. Bump the Roslyn pin in `Directory.Packages.props`.
2. **Wire the post-init pipeline and remove the writer's decoration helper.** Update `ServiceRegistrationGenerator.Initialize`, delete `GenerateDecorationHelper` + `DecorationHelperBody`, drop the conditional `Injectio.Decoration.g.cs` emit. Refresh test harness selectors and snapshots in this same commit.
3. **Switch packaging to Generators.** Update `Injectio.Generators.csproj` per §5; mark `Injectio.Attributes.csproj` non-packable and strip its `PackageId` + packing items.
4. **Remove project references and delete the Attributes project.** Update the four test/sample csproj files. Delete `src/Injectio.Attributes/` and remove its entry from `Injectio.slnx`.
5. **README + version-floor doc updates.**
